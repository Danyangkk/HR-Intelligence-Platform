**Bug ID**: BUG-002  
**报告日期**: 2026-05-28 16:37

## 📋 问题原话

> Bug：薪资二次确认弹窗触发点绑错位置——"首次进入薪资区"没弹，"切 Tab"才弹。
> 现象：业务超管点左侧导览 L2 薪资分类（如"奖金"）进入时，自动加载第一张表数据，此时不弹确认窗；手动切换顶部 L3 Tab（如"月度奖金发放明细表"）时才弹窗。

---

## 🔍 问题现象

### 触发场景
- **步骤1**: 业务超管（biz_hrd）登录后
- **步骤2**: 点击左侧导览"薪酬"大类下的L2分类（如"奖金"）
- **预期行为**: 立即弹出"薪资数据访问确认"弹窗
- **实际行为**: ❌ 无弹窗，直接加载第一张表数据
- **步骤3**: 手动切换顶部L3 Tab（如"月度奖金发放明细表"）
- **实际行为**: ✅ 此时才弹出确认窗

### 规格要求
- **粒度B**: 业务超管本会话**首次访问薪资区**时弹一次确认，之后本会话内所有薪资数据请求（切表/翻页/筛选/换L2薪资子类）都不再弹
- **触发点**: "发起薪资数据请求"的入口，而不是某个UI交互（L3 Tab切换）
- **参考**: 前端页面规格-权限重构.md §3.3 二次确认弹层

## 🎯 根因分析

### 代码路径分析

#### 路径1：点击L2薪资分类（有Bug）
```javascript
用户点击L2"奖金"
 ↓
selectL2(l2id)  // Line 1467
 ↓
state.activeL3 = l2.children[0].id  // 自动设置第一张L3表
 ↓
renderAll()  // 直接加载数据
 ↓
❌ 无薪资检查，直接加载！
```

#### 路径2：点击L3 Tab切换（正常）
```javascript
用户点击L3 Tab"月度奖金发放明细表"
 ↓
selectL3(id)  // Line 1501
 ↓
isPayroll = isPayrollL3(id)  // 检查是否薪资数据
 ↓
if(isPayroll) ensurePayrollAccess()  // ✅ 二次确认
 ↓
renderAll()
```

### 直接原因
- **弹窗逻辑只绑在了`selectL3`**（L3 Tab切换）上
- **`selectL2`函数缺少薪资检查**，直接设置`state.activeL3`并加载数据

### 根本原因
- **触发位置设计错误**: 弹窗逻辑绑在UI交互（L3 Tab切换）上，而不是"发起薪资数据请求"的统一入口
- **会话粒度缺失**: 当前使用`payrollConfirmToken`（单次有效），不符合"本会话首次确认后不再弹"的规格要求

---

## ✅ 修复方案

### 架构改进
**统一收口**: 把薪资数据请求的二次确认逻辑收口到`ensurePayrollAccess`函数，使用会话标记控制粒度。

### 代码修改

#### 1. 添加会话标记
```javascript
// frontend/index.html - AUTH对象
const AUTH = {
  // ...
  payrollConfirmedThisSession: false,  // 本会话是否已确认薪资访问（粒度B）
};
```

#### 2. 修改 `ensurePayrollAccess` - 实现"本会话首次确认"
```javascript
// frontend/permission-admin.js
async function ensurePayrollAccess(ctx){
  if(global.AUTH.role !== 'biz_super_admin') return false;
  
  // 粒度B：本会话已确认，直接放行（不再弹窗）
  if(global.AUTH.payrollConfirmedThisSession) return true;
  
  // 首次访问薪资，弹出二次确认
  const ok = await openPayrollConfirmModal(ctx);
  if(ok){
    global.AUTH.payrollConfirmedThisSession = true;  // 标记本会话已确认
  }
  return !!ok;
}
```

#### 3. 修改 `selectL2` - 自动加载第一张薪资表前先确认
```javascript
// frontend/index.html
async function selectL2(l2id){
  hideBackFloat();
  for(const l1 of categoryTree){
    for(const l2 of l1.children){
      if(l2.id === l2id){
        state.expandedL1.add(l1.id);
        state.activeL2 = l2id;
        
        // 新增：检查第一张L3是否是薪资数据
        const firstL3 = l2.children[0].id;
        const isPayroll = typeof isPayrollL3==='function' && isPayrollL3(firstL3);
        
        if(isPayroll){
          if(AUTH.role === 'biz_super_admin'){
            const ok = await ensurePayrollAccess({
              target_ref: '数据中台查询',
              entry: l2.name
            });
            if(!ok) return;  // 用户取消，不加载
          } else {
            toast('无权访问薪资数据', 'danger');
            return;
          }
        }
        
        state.activeL3 = firstL3;
        resetView();
        void renderAll();
        return;
      }
    }
  }
}
```

#### 4. 登出时重置会话标记
```javascript
// frontend/permission-admin.js
function authLogout(){
  // ...
  global.AUTH.payrollConfirmedThisSession = false;  // 重置会话标记
  // ...
}
```

## 📚 项目经验更新

### 新增到 `.sdd/experience.md`

#### 薪资二次确认触发逻辑（规格粒度B）

**规格要求**:
- ✅ **触发粒度B**: 业务超管本会话**首次访问薪资区**时弹一次确认
- ✅ **会话内不再弹**: 本会话内所有薪资数据请求（切表/翻页/筛选/换L2薪资子类）都不再弹
- ✅ **登出重置**: 重新登录后，首次访问薪资再次弹窗

**实现要点**:
1. **会话标记**: `AUTH.payrollConfirmedThisSession`（布尔值）
2. **触发收口**: 所有会加载薪资数据的路径都调用统一的`ensurePayrollAccess`
   - 点击L2薪资分类（`selectL2`）→ 自动加载第一张L3表前检查
   - 切换L3 Tab（`selectL3`）→ 加载前检查
   - 智能体薪资查询（`handleAgentEvent`）→ 生成答案前检查
3. **登出重置**: `authLogout`函数中重置会话标记

**错误案例**:
- ❌ **只在L3 Tab切换时检查**: 会导致"点L2进入薪资区"无弹窗
- ❌ **使用单次token而非会话标记**: 会导致每次请求都弹窗，不符合粒度B

**正确路径**:
```
点L2薪资分类 → selectL2 检查第一张L3 → ensurePayrollAccess
切L3 Tab → selectL3 → ensurePayrollAccess
智能体薪资查询 → reject"需要二次确认" → handlePayrollConfirmForAgent → ensurePayrollAccess
```

**相关文件**:
- `frontend/index.html` - `selectL2`, `selectL3`, `handleAgentEvent`, `AUTH`对象
- `frontend/permission-admin.js` - `ensurePayrollAccess`, `authLogout`
