## 📋 用户反馈

### 问题1：显示层级错误
弹窗中"你即将查看："显示的是 **L3表名**（如"员工通知记录"），但用户期望显示的是 **L2分类名**（如"员工通知记录" 的上级分类"员工关系"）。

**原因**：薪资数据组织为三层：
- L1: 薪酬（大类）
- L2: 员工通知记录、工资、奖金等（二级分类）
- L3: 具体表名（如"月度工资表"、"月度奖金发放明细表"）

用户希望弹窗显示的是 **二级分类名称**，更符合用户心智模型。

### 问题2：事由输入形式不合理
事由使用单行 `<input>` 输入框，但合规审计事由通常需要更详细的说明，单行输入不够友好。

## ✅ 修复内容

### 一、修改弹窗HTML结构

**文件**: `frontend/index.html`

**修改前**:
```html
<label>访问事由<input id="payrollConfirmReason" type="text" placeholder="必填，用于合规审计"></label>
```

**修改后**:
```html
<label>访问事由<textarea id="payrollConfirmReason" rows="3" placeholder="必填，用于合规审计"></textarea>
```

**效果**:
- ✅ 从单行输入框改为3行文本框
- ✅ 用户可以输入更详细的访问事由
- ✅ 更符合合规审计要求

### 二、修改 `selectL2` 传参

**文件**: `frontend/index.html`

**修改前**:
```javascript
const ok = await ensurePayrollAccess({
  target_ref: firstL3Name,  // L3表名（如"薪资核算表"）
  entry: '数据中台',
  fields: '薪资'
});
```

**修改后**:
```javascript
const ok = await ensurePayrollAccess({
  target_ref: l2.name,  // L2分类名称（如"员工通知记录"）
  entry: '数据中台',
  fields: '薪资'
});
```

**说明**:
- 点击 L2 分类（如"工资"）进入时，弹窗显示 **"工资"**（L2名称）
- 而不是显示 **"月度工资表"**（第一张L3表的名称）

### 三、修改 `selectL3` 传参

**文件**: `frontend/index.html`

**修改前**:
```javascript
const ok = await ensurePayrollAccess({
  target_ref: l3Name(id),  // L3表名（如"月度奖金发放明细表"）
  entry: '数据中台',
  fields: '薪资'
});
```

**修改后**:
```javascript
// 找到当前L3所属的L2分类名称
let l2Name = '';
for(const l1 of categoryTree){
  for(const l2 of l1.children){
    if(l2.children && l2.children.some(l3 => l3.id === id)){
      l2Name = l2.name;
      break;
    }
  }
  if(l2Name) break;
}

const ok = await ensurePayrollAccess({
  target_ref: l2Name || l3Name(id),  // L2分类名称，如果找不到则用L3表名兜底
  entry: '数据中台',
  fields: '薪资'
});
```

**说明**:
- 切换 L3 Tab（如从"月度工资表"切到"月度奖金发放明细表"）时，弹窗显示 **"奖金"**（L2分类）
- 而不是显示 **"月度奖金发放明细表"**（L3表名）
- 增加了兜底逻辑：如果找不到L2（理论上不应该），则显示L3表名

## 📝 技术说明

### 查找L2分类的算法

```javascript
// 遍历三层树结构，找到L3所属的L2
let l2Name = '';
for(const l1 of categoryTree){
  for(const l2 of l1.children){
    if(l2.children && l2.children.some(l3 => l3.id === id)){
      l2Name = l2.name;  // 找到了
      break;
    }
  }
  if(l2Name) break;
}
```

**时间复杂度**: O(L1 × L2 × L3)，在实际使用中很小（约 4 × 5 × 3 = 60次比较）

**兜底机制**: 如果找不到L2（理论上不应该发生），则显示L3表名作为兜底。
