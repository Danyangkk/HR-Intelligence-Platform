## 📋 问题描述

### 原有问题
1. **工单状态显示英文码**：`pending`, `in_progress`, `awaiting_gate`, `rejected` 等直接显示
2. **rejected状态误显示**：已驳回工单的操作列显示灰色文字 "rejected"，应为空
3. **其他枚举值未统一**：角色、审计入口等枚举值部分地方显示英文

### 用户体验问题
- ❌ 普通用户看到 `biz_super_admin`、`pending` 等机器码，不友好
- ❌ 已驳回工单显示多余文字，UI不整洁
- ❌ 没有统一的枚举值映射规范

---

## ✅ 修复内容

### 一、添加枚举映射函数

**文件**: `frontend/permission-admin.js`

```javascript
// 工单状态中文映射
const TICKET_STATUS = {
  pending: '待处理',
  in_progress: '处理中',
  awaiting_gate: '待验证',
  done: '已上线',
  rejected: '已驳回',
  hold: '存疑',
};
function ticketStatusLabel(status){
  return TICKET_STATUS[status] || status;
}

// 审计入口中文映射
const ENTRY_LABELS = {
  '数据中台': '数据中台',
  '智能体': '智能体',
  'data': '数据中台',
  'agent': '智能体',
};
function entryLabel(entry){
  return ENTRY_LABELS[entry] || entry;
}
```

### 二、修改"我的工单"页面

**修改前**:
```javascript
<td>${global.escAi(t.status)}</td>  // 显示 "pending"、"rejected" 等
const ops = ...: `<span class="muted">${global.escAi(t.status)}</span>`  // rejected 显示文字
```

**修改后**:
```javascript
<td>${ticketStatusLabel(t.status)}</td>  // 显示 "待处理"、"已驳回" 等
const ops = t.status==='rejected'?'': ''  // rejected 不显示任何内容
```

### 三、修改"改进工单追踪"页面

**修改前**:
```javascript
<td>${global.escAi(t.status)}</td>  // 显示英文
```

**修改后**:
```javascript
<td>${ticketStatusLabel(t.status)}</td>  // 显示中文
```

### 四、修改"薪资访问审计"页面

**修改前**:
```javascript
<td>${global.escAi(r.entry)}</td>  // 可能显示英文 "data"、"agent"
```

**修改后**:
```javascript
<td>${entryLabel(r.entry)}</td>  // 显示 "数据中台"、"智能体"
```

### 五、修改工单详情弹窗

**修改前**:
```javascript
alert(`工单 ${t.ticket_no}\n...\n状态：${t.status}\n...`);  // 显示英文
```

**修改后**:
```javascript
alert(`工单 ${t.ticket_no}\n...\n状态：${ticketStatusLabel(t.status)}\n...`);  // 显示中文
```

## 📝 规范说明

### 枚举值映射原则

1. **统一映射函数**：所有枚举值都应有对应的映射函数（如 `ticketStatusLabel`）
2. **后端优先**：优先使用后端返回的 `*_label` 字段（如 `role_label`）
3. **前端兜底**：如果后端没有返回中文，前端用映射函数转换
4. **不显示英文码**：任何情况下都不要直接显示 `pending`、`biz_hrd` 等机器码

### 未来新增枚举类型时

1. 在 `permission-admin.js` 顶部添加常量和映射函数
2. 在所有显示该枚举值的地方使用映射函数
3. 不要直接 `${data.enum_field}`，而是 `${enumLabel(data.enum_field)}`
