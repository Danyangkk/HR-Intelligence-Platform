### 2. 修复前端循环保护机制

#### 问题根因

用户报告已经 55 次循环。分析发现：

每次 `runAgentStream` 都会创建**全新的 msg 对象**：

```javascript
const msg={role:'ai',mid,stage:0,done:false,...};  // 新对象，没有 payrollConfirmRetry
```

导致：
1. 第一次 reject → `msg.payrollConfirmRetry = 1` → 触发确认
2. 确认后调用 `runAgentStream` → 创建**新 msg**（`payrollConfirmRetry` 丢失）
3. 新的 reject 事件 → `msg.payrollConfirmRetry` 又从 undefined 变为 1 → 再次触发
4. 无限循环...

#### 修复方案

**将 `payrollConfirmRetry` 计数器从 msg 对象移到全局 `AI` 对象**，确保跨多次 `runAgentStream` 调用时计数器不丢失。

**文件**: `frontend/index.html`

##### 修改点 1: reject 事件处理

```javascript
}else if(event==='reject'){
  if(data.reason === '需要二次确认'){
    // 🔥 使用全局计数器（因为每次runAgentStream都创建新msg对象）
    if(!AI.payrollConfirmRetry) AI.payrollConfirmRetry = 0;
    AI.payrollConfirmRetry++;
    
    console.log('[reject事件] 需要二次确认，全局重试次数:', AI.payrollConfirmRetry);
    
    // 循环保护：如果已经在处理确认，不要重复触发
    if(msg.needsPayrollConfirm){
      console.error('[循环保护] 检测到重复的薪资确认请求，已拦截');
      msg.answer=`<div class="ans-reject"><b>薪资确认失败</b><br>Token传递异常，请刷新页面重试。</div>`;
      msg.done=true; msg.orchOpen=false;
      AI.payrollConfirmRetry = 0; // 重置计数器
      return;
    }
    
    // 检查重试次数（最多重试1次，即总共2次尝试）
    if(AI.payrollConfirmRetry > 2){
      console.error('[循环保护] 薪资确认重试次数超限，已累计:', AI.payrollConfirmRetry, '次');
      msg.answer=`<div class="ans-reject"><b>薪资确认失败</b><br>后端Token验证异常，已累计 ${AI.payrollConfirmRetry} 次确认请求。请查看后端日志定位问题。</div>`;
      msg.done=true; msg.orchOpen=false;
      AI.payrollConfirmRetry = 0; // 重置计数器
      return;
    }
    
    // 触发二次确认弹窗
    msg.answer=`<div class="ans-reject"><b>薪资数据访问需要二次确认</b><br>请在弹窗中填写访问事由后查看。</div>`;
    msg.needsPayrollConfirm = true;
    msg.originalQuestion = q;
    msg.done=true; msg.orchOpen=false;
    setTimeout(() => handlePayrollConfirmForAgent(s, msg), 100);
  } else {
    // 其他 reject 原因...
  }
}
```

##### 修改点 2: 成功答案后重置计数器

```javascript
}else if(event==='answer'){
  // ... 处理答案 ...
  
  // 成功收到答案，重置薪资确认重试计数器
  if(AI.payrollConfirmRetry > 0){
    console.log('[answer事件] 成功收到答案，重置薪资确认重试计数器');
    AI.payrollConfirmRetry = 0;
  }
}
```

##### 修改点 3: 取消确认时重置计数器

```javascript
async function handlePayrollConfirmForAgent(s, msg){
  // ... 处理确认 ...
  
  if(ok && AUTH.payrollConfirmToken){
    // 确认成功，重新发送请求（不重置计数器，以便触发循环保护）
    runAgentStream(s, msg.originalQuestion);
  } else {
    // 用户取消确认
    msg.answer = '<div class="ans-reject"><b>已取消薪资数据访问</b><br>你可以重新提问。</div>';
    msg.needsPayrollConfirm = false;
    AI.payrollConfirmRetry = 0;  // ← 重置计数器
    renderAiThread();
  }
}
```

#### 循环保护逻辑

1. **第 1 次 reject** → `AI.payrollConfirmRetry = 1` → 弹窗确认
2. **确认后重发** → 如果再次 reject → `AI.payrollConfirmRetry = 2` → 弹窗确认
3. **再次重发** → 如果第三次 reject → `AI.payrollConfirmRetry = 3` (>2) → **停止并显示错误**："后端Token验证异常，已累计 3 次确认请求。请查看后端日志定位问题。"

这样即使后端验证有问题，前端最多尝试 2 次，不会无限循环。
