#### 新增函数: `handlePayrollConfirmForAgent()`
```javascript
async function handlePayrollConfirmForAgent(s, msg){
  // 打开薪资二次确认弹窗（智能体入口）
  const ok = await ensurePayrollAccess({
    target_ref: '智能体查询',
    entry: '个人薪资明细'
  });
  
  if(ok && AUTH.payrollConfirmToken){
    // 确认成功，重新发送请求
    msg.answer = '<div class="ai-wait">已确认，正在生成答案…</div>';
    msg.done = false;
    renderAiThread();
    // 重新运行agent，这次带上确认token
    runAgentStream(s, msg.originalQuestion);
  } else {
    // 用户取消确认
    msg.answer = '<div class="ans-reject"><b>已取消薪资数据访问</b><br>你可以重新提问。</div>';
    renderAiThread();
  }
}
```

**流程**:
1. 业务超管问薪资问题
2. 后端Planner检测到 → 返回 reject事件，reason="需要二次确认"
3. 前端检测到 → 弹出`payrollConfirmModal`弹窗
4. 用户填写事由并确认 → 获得`confirm_token`
5. 前端重新发送请求，headers带上`X-Payroll-Confirm: <token>`
6. 后端放行 → 正常生成答案 + 写入审计日志
