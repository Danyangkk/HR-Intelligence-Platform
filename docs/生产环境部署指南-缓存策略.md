# 生产环境部署指南 - 缓存策略

## 📌 当前配置（开发/测试环境）

**文件**: `nginx/nginx.conf`

```nginx
# HTML: 5分钟缓存
location ~* \.html$ {
    add_header Cache-Control "public, max-age=300, must-revalidate";
}

# JS/CSS: 10分钟缓存
location ~* \.(js|css)$ {
    add_header Cache-Control "public, max-age=600";
}

# 静态资源: 1年缓存
location ~* \.(jpg|jpeg|png|gif|ico|svg|woff|woff2|ttf|eot)$ {
    add_header Cache-Control "public, max-age=31536000, immutable";
}
```

**说明**：
- ✅ **不会影响功能** - 浏览器会定期刷新
- ✅ **开发友好** - 5-10分钟后自动获取新版本
- ⚠️ **性能一般** - 每5-10分钟会重新请求文件

---

## 🚀 生产环境推荐配置

### 方案A: 使用文件名Hash（最佳实践）⭐

**原理**: 使用构建工具（Webpack/Vite/Rollup）自动给JS/CSS文件名加hash

```
index.html
├── app.abc123.js      ← 文件名包含内容hash
├── vendor.def456.js
└── styles.789xyz.css
```

**nginx配置**：`nginx.conf.production`（已创建）

```nginx
# HTML: 1小时缓存，需验证
location ~* \.html$ {
    add_header Cache-Control "public, max-age=3600, must-revalidate";
    etag on;
}

# 带hash的JS/CSS: 永久缓存（1年）
location ~* \.(js|css)$ {
    add_header Cache-Control "public, max-age=31536000, immutable";
    etag on;
}
```

**优点**：
- ✅ 极佳性能 - JS/CSS永久缓存，只下载一次
- ✅ 即时更新 - 代码变化时文件名改变，自动失效旧缓存
- ✅ CDN友好 - 可配合CDN实现全球加速

**实现步骤**：
1. 使用构建工具（推荐Vite）打包项目
2. 配置输出文件名带hash
3. HTML自动引用带hash的文件
4. 部署时替换nginx配置为 `nginx.conf.production`

---

### 方案B: 不改代码（当前项目适用）

**如果不想改构建流程**，可以调整缓存时间：

#### 开发/测试环境（当前配置）：
```nginx
# HTML: 5分钟
# JS/CSS: 10分钟
```

#### 生产环境（调整后）：
```nginx
# HTML: 1小时
location ~* \.html$ {
    add_header Cache-Control "public, max-age=3600, must-revalidate";
    etag on;
}

# JS/CSS: 1小时
location ~* \.(js|css)$ {
    add_header Cache-Control "public, max-age=3600";
    etag on;
}
```

**权衡**：
- ✅ 性能提升 - 减少80%+的重复请求
- ⚠️ 更新延迟 - 用户可能需要等1小时才能看到更新
- 💡 **解决方案**: 每次发版后通知用户刷新，或在版本更新弹窗中提示用户 `Ctrl+F5` 强制刷新

## 🔄 如何在生产环境应用

### 部署时切换配置

```bash
# 生产环境部署时
cd /your/production/path
cp nginx/nginx.conf.production nginx/nginx.conf
docker compose restart nginx
```

### 或使用环境变量切换

修改 `docker-compose.yml`:

```yaml
nginx:
  volumes:
    - ./frontend:/usr/share/nginx/html:ro
    - ./nginx/nginx.conf.${ENV:-dev}:/etc/nginx/conf.d/default.conf:ro
```

然后：
```bash
# 开发环境
ENV=dev docker compose up

# 生产环境
ENV=production docker compose up
```

## ✅ 总结

### 对于当前项目（HR-Agent）：

1. **开发/测试环境** → 继续使用当前配置（5-10分钟缓存）
   - ✅ 已配置完成，无需改动
   - ✅ 不影响开发调试

- ✅ 只影响开发环境
