# Feature: 薪资确认30分钟TTL + 审计表字段修正

**日期**: 2026-05-28 17:42

## 📋 需求概述

### 一、薪资确认改为30分钟TTL（账号级，两入口共用）

**原有问题**:
- 会话级标记（刷新页面需要重新确认）
- 数据中台和智能体不共用确认状态

**新方案**:
- **30分钟TTL**：确认后30分钟内访问薪资无需重新确认
- **两入口共用**：数据中台确认后，智能体也在TTL内免弹；反之亦然
- **登出重置**：登出后TTL失效，重新登录需重新确认

### 二、审计表字段修正（合规要求）

**原有问题**:
1. 访问人显示角色代码（如`biz_hrd`），无法追溯到具体人
2. 智能体查询"访问对象"和"字段"都写"个人薪资明细"，未记录被查员工
3. 数据中台"访问对象"和"字段"重复（都写"工资"）

**新方案**:
| 字段 | 数据中台入口 | 智能体入口 |
|------|-------------|-----------|
| 访问人 | 张HRD（HR0001） | 张HRD（HR0001） |
| 访问对象 | 表名（月度工资表） | 被查员工（张三 A0123） |
| 入口 | 数据中台 | 智能体 |
| 字段 | 实际访问的薪资字段 | 查询的薪资字段（奖金/工资） |
| 事由 | 用户填的 | 用户填的 |

**红线**：永不记录薪资数值本身

#### 2. ensurePayrollAccess - 检查TTL而非布尔值

**文件**: `frontend/permission-admin.js`

```javascript
// 修改前
if(global.AUTH.payrollConfirmedThisSession) return true;

// 修改后

if(global.AUTH.payrollConfirmedUntil > now){
  return true;  // TTL内，免弹
```

### 后端修改（2处）

#### 1. log_payroll_access - 格式化访问人和参数

**文件**: `backend/src/services/payroll_access.py`

```python
async def log_payroll_access(
    db: AsyncSession,
    *,
    actor: str,  # username
    actor_display: str,  # 显示名称（姓名）
    employee_id: str | None,  # 工号
    target_ref: str,  # 访问对象：数据中台=表名，智能体=被查员工
    entry: str,  # 入口：数据中台 / 智能体
    fields: str,  # 实际访问的薪资字段
    reason: str,
) -> dict[str, Any]:
    # 格式化访问人：姓名（工号）
    if employee_id:
        formatted_actor = f"{actor_display}（{employee_id}）"
    else:
        formatted_actor = actor_display or actor
    
    row = PayrollAccessLog(
        actor=formatted_actor,  # 显示为"姓名（工号）"
        target_ref=target_ref,
        entry=entry,
        fields=fields,
        reason=reason.strip(),
    )
    db.add(row)
    await db.commit()
    return {"logged": True, "id": row.id}
```

#### 2. create_confirm_token - 调用新签名

**文件**: `backend/src/services/payroll_access.py`

